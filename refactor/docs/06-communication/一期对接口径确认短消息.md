# 一期对接口径确认短消息

您好，我们整理了一版一期对接默认口径，主要用于先统一任务、设备、结果、报告和文件交付的理解。

如无异议，我们后续会按以下口径推进；如有不一致的地方，请直接指出需要调整的部分。

1. 我们按完整二级子系统接入，不按单个设备或单个执行节点处理。
2. `deviceId` 按业务设备身份理解；串口、网口、SCOE 通道属于我们内部连接对象。
3. `setTestTask` 是任务下发，`startTestCaseList` 是任务启动。
4. 一期按单任务串行执行；已有任务运行时，新任务或再次启动先按不可执行处理。
5. 一期控制动作按 `stop` 处理，stop 后任务状态按 `stopped` 处理。
6. stop 后仍生成 JSON 报告，记录 stopped、已完成 case 和未完成 case。
7. case result 和 task result 分开处理。
8. JSON 报告生成、文件传输完成、业务接收完成分开判断。
9. 结果和报告时间先按我们系统时间记录；如需统一时区或固定格式，请提供规则。
10. 一期先按基础状态和心跳对接。

需要确认或提供的内容主要有：

1. 子系统和设备标识的分配方式，以及多个内部设备如何对应 `deviceId`。
2. `startTestCaseList` 一期是否可以按启动当前任务内 case 执行处理。
3. 一期单任务串行执行、重复任务按重复请求处理是否可以接受。
4. `stop` 是否是一期控制动作。
5. case result、task result 的最小分类、错误码和原因说明。
6. TestReport 是否就是 JSON 报告文件、一期必须包含的内容，以及一份报告示例。
7. 报告文件名、目录、交付方式；如需要上传，请提供测试环境、账号和目录权限。
8. 心跳是无响应，还是需要返回 `heartbeatResponse`。
9. Swagger / OpenAPI、测试环境地址、鉴权方式和测试账号。
