## 七、字段审查维度

请不要按“字段说明”写，而是对每个字段都尽量回答以下问题：

1. `文档当前怎么写`
   - 只写可回溯事实，不写推测。

2. `本轮可能怎么用`
   - 这个字段是否真的进入当前主线。

3. `我方是否有稳定数据来源`
   - 有、没有、部分有、要靠默认值 / 透传。

4. `能否降级处理`
   - 是否可以默认、透传、空置、后置，不先进入内部主逻辑。

5. `甲方可能怎么拿它卡人`
   - 例如把示例字段说成必填、把展示字段说成验收字段、把扩展字段说成本轮前置。

6. `风险判断`
   - 这个字段会不会让我后续实现、联调、沟通或扯皮时吃亏。

7. `建议处理方式`
   - `本轮真接`
   - `先默认`
   - `先透传`
   - `先挂空`
   - `后置`
   - `必须和甲方确认`

8. `被追问时最省事的回应口径`
   - 用一句话写清楚，如果甲方硬卡，我方最省事怎么回。
