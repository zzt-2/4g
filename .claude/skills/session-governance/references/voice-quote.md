# Voice Quote — 用户原话记录规范

> 配合每个专题的 `voice.md`。极简原则：原话占主体，元数据压到最小。
> 提炼自 `2026-06-04-advisor-review-revision/PROMPT-001` 的 `来源: 批注 N "原话"` 模式。

## 核心原则

voice.md 是**用户/导师原话的忠实档案**，不是分析报告。原话占绝对主体，Claude 的加工（态度标签/归纳/语境注释）趋近于零。

**禁止膨胀**：不为一条原话加数倍的说明。一条原话就是原话本身，最多带一个可选的产出链接。

## 收录边界

- **全收**：除零信息推进/应答外的所有带态度/信息/反馈的原话，包括低密度的（"再想想""还行""这个可以看看"）。**不去重**（同一诉求多次说，各自保留独立行）。
- **不收**：零信息纯推进/应答（"继续""嗯""好""对""ok""下一步""继续做"）、纯操作指令（"跑测试""读文件"）。

## 格式（极简）

每条原话一行，按日期归组：

```
## YYYY-MM-DD
- "原话" → D###
- "另一句原话"
```

元数据只在这些情况加（其余一律不加）：

- `→产出`：原话触发/关联了某 D###/S###/H### 时标，就 `→ D001`，不解释。无对应产出则不标。
- `[转述]`：原话来自 session note 的转述而非 verbatim，前置标注。
- `来源`：仅当不是用户本人（导师/批注/他人）时标，如 `来源:批注12`。用户说的不标。
- `⟶`：仅当与此前原话态度对立时标冲突（见下）。

## 长原话

直接放，或用 blockquote，**不加语境注释行**：

```
- > "这是一段较长的原话，完整保留……" → D###
```

## 冲突标记（仅对立时，不每条加）

当新原话与旧原话态度/立场对立：

- 旧行加 `⟶ 被 YYYY-MM-DD 推翻`
- 新行加 `⟶ 推翻 YYYY-MM-DD`

沿用 decisions.md 的 superseded 思想：旧的不删，保留演变轨迹。只在立场反转/约束推翻时标，细微波动不标。

## 完整示例

虚构专题，展示所有形态（短句/无产出/转述/长原话/批注来源/冲突双向）：

```markdown
# Voice — paper-direction-x

> 用户/导师原话档案，按日期。除零信息推进/应答外都收，不去重。

## 2026-06-10
- "大空白不是不好做就是有问题" → D001排除
- "方向太少"
- [转述]"之前方向太少，容易找不到合适的"
- > "这段我之前走过，大空白看着诱人但十个是坑，先稳一点" → D001 C1
- 来源:批注5 "第二章逻辑断裂，重写"

## 2026-06-14
- "ML 一定要看" → D001 ML优先
- ⟶ "光扩展太保守，这次想试试原创方向" 推翻 2026-06-10

## 2026-06-20
- ⟶ "还是回到扩展型，原创风险太大" 推翻 2026-06-14
```

## 归档（物理后手，不加说明）

voice.md 超 ~300 行（或跨多阶段）时，把**最早的批次整体移到 `voice-archive.md`**，主文件保留近期。归档卷顶部放一行指向说明。原话一条不丢，只是分文件。

## 与 User Attitude 段落的关系

全局 CLAUDE.md 恢复规范的 `User Attitude / 用户态度` 段落（压缩恢复时的临时输出）从 `voice.md` 取原话。voice.md 是持久源，User Attitude 是临时投影。

## 创建时机

专题首次产生符合收录边界的原话时创建 `voice.md`，不预建空文件。
