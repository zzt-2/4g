# V### Verification Record Template

每个专题的 `verifications.md` 中，每条 V### 条目使用以下格式。

## 标准格式

```markdown
## V###: [验证标题]

> date: YYYY-MM-DD
> 关联：S### / D###

### 验证项

- [ ] 项1：[方法] → [结果]
- [ ] 项2：[方法] → [结果]
- [ ] 项3：[方法] → [结果]

### 证据

[命令行输出、测试结果等。粘贴原始数据，不要概括]

### 结论

PASS / FAIL / PARTIAL

### 后续（FAIL/PARTIAL 时）

[需要做什么来修复]
```

## 字段说明

- **验证项**: 每项必须有方法（怎么验证）和结果（发现了什么）。用复选框。
- **证据**: 支持结论的原始数据。命令输出、测试结果、diff 等。不要概括——粘贴实际证据。
- **结论**: 必须是 PASS / FAIL / PARTIAL 三选一。不用"基本通过"、"看起来还行"。
- **后续**: 仅 FAIL/PARTIAL 时填写。需要什么具体步骤来修复。

## 何时创建 V###

- 实现了 S### 或 D### 中描述的功能或修复后
- 运行测试来验证一个决策后
- 在宣称任务完成之前
- handoff 验证发现问题时
- 路线验证（验证某条路线是否可行）

## 验证与 Session Note 的关系

- Session note 记录过程和讨论
- Verification record 记录基于证据的验证
- 两者分离：不自审自验（不在同一个文档中既实施又验证）
- V### 引用 S### 或 D###，S### 引用 V### 的结论
