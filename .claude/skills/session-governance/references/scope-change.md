# Scope Change Record Format

范围变更时，在 topic-index.md 的"范围变更记录"段落中新增条目。

## 格式

```markdown
- **[YYYY-MM-DD]** [决策D###]：[变更内容]
  - 原因：[为什么需要变更]
  - 新范围：[变更后的范围描述]
  - 影响的未决项：[哪些未决项因此变更]
```

## 何时必须记录

以下情况**必须**记录 scope change：

1. 当前 session 的工作超出了 topic-index 中的"当前范围"
2. 新增了原始计划中没有的任务
3. 触及了"明确不含"列表中的内容
4. 专题目标偏离了原始意图

以下情况**不需要**记录：

1. 当前范围内的正常推进
2. 对现有范围的澄清（不是扩大）
3. 发现原本就隐含的子任务

## 阻断规则

- **触及"明确不含"** → 阻断：要求用户显式确认 + scope change record + D### 决策
- **专题 >= 8 sessions** → 提醒：建议检查范围是否仍在原始目标内
- **专题 >= 15 sessions** → 阻断：scope change record 为强制性，不记录不允许继续

## topic-index 范围边界模板

新建专题时，topic-index.md 必须包含范围边界段落：

```markdown
## 范围边界

**原始目标**：[不可修改，冻结记录]
**当前范围**：[可随 scope-change 决策更新]
**明确不含**：[显式排除项]
**范围变更记录**：
- （无变更时留空）
```

### 不变量段落

在"已确认结论"下增加不变量子段落：

```markdown
## 已确认结论

### 不变量（动任何一条必须重新讨论）
1. ...
2. ...

### 其他结论
1. ...
```

- 不变量 = 优先级最高的结论，架构约束级别
- 变更不变量必须回 topic-index 显式修改，不能在 S### 里悄悄推翻
