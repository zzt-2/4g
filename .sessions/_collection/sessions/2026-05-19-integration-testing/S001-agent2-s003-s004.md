### Q3: S004 Task Service 层实现的 bug 修复与集成缺陷暗示

**来源**：S004 "二、Task Service/State/Selectors 层实现" + "四、Task 修复链"

S004 记录的 Task Service 实现 bug 共 6 项（S004 "二，实现过程修复"）：

| # | Bug | 详情 | 是否暗示集成缺陷 |
|---|-----|------|-----------------|
| 1 | selectTaskHistory 缺少导入 | 缺少 import 语句 | **是** — selector 暴露了但引用链不完整，说明模块拆分后集成验证不充分 |
| 2 | retry 递归错误 | retry 应调原始 step executor 而非自身 | **是** — 循环引用/递归逻辑在 service 聚合后才暴露，单元测试可能只覆盖了 core 层 |
| 3 | 非触发模式缺少 receive event 订阅 | 某些模式下未订阅 receive 事件 | **是** — task 与 receive 的跨 feature 交互在 service 层装配时才显现缺失 |
| 4 | settle 函数未检查 paused 状态 | settle 循环不检查暂停标志 | **是** — lifecycle 状态与执行循环的状态同步是典型的集成缺陷模式 |
| 5 | trigger 循环迭代间丢事件竞态 | 触发器循环在迭代间丢失事件 | **是** — 并发/竞态问题只在 service 层聚合后才可观察 |
| 6 | retry 测试中 500ms delay 超时 | 测试用例中 delay 导致超时 | **否** — 测试基础设施问题，非业务逻辑缺陷 |

**进一步修复链**（S004 "四"）：

- stopCondition 合并 — 待确认是否属于 bug 修复还是设计变更
- task-service 731 行拆成 6 模块 — 重构，间接说明初始实现集成度不够、模块边界不清
- shared 层提取 cloneUnknownValue + defaultNow + compareValues — 多 feature 重复实现发现，说明跨 feature 集成时才发现共享需求
- as 断言修复 — 类型安全问题

**判断**：6 个 bug 中 5 个是明确的集成缺陷（在 feature core 通过单测后、service 层聚合才暴露）。这证实 task-service 层是集成风险集中区域。

### Q5: S004 架构文档 34 处同步的文档-代码不一致风险

**来源**：S004 "十、架构文档同步"

**事实**：

- 4 个并行子 agent 编辑 4 份文档，共 34 处修改
- 文档列表：
  - `rewrite-target-structure.md` — 7 处（task 系统定义、feature 表 4 行、边界例外、放置示例）
  - `rewrite-system-architecture.md` — 4 处（feature 表 4 行）
  - `rewrite-feature-boundaries.md` — 8 处（边界表 4 行 + owner notes 4 节）
  - `rewrite-feature-interaction-matrix.md` — 15 处（矩阵行 8 + 风险 2 + 禁止项 1 + pre-design gates 4）
- 核心变更：task 从"独立能力域"升级为"通用执行引擎"，send 缩窄为单帧发送，新增 SCOE/指令接入 feature

**风险评估依据**：

1. **并行编辑无交叉审查证据** — 4 个子 agent 并行编辑，S004 未记录子 agent 间是否有交叉验证
2. **34 处修改散布在 4 份文档** — 任何一处遗漏或不一致都会导致后续设计/实施基于错误文档
3. **已有先例**：S004 "九"中 TransportEventSnapshot 缺 bytes 字段是设计文档与代码不匹配的实例，说明文档-代码 drift 已实际发生过
4. **核心语义变更**（task 角色升级）涉及所有消费 task 的 feature，如果文档间对 task 的描述不完全一致，会造成理解分歧
5. **interaction matrix 15 处修改量最大**，矩阵的行列交叉引用容易出现遗漏

**待确认项**：

- 是否有 post-sync 验证步骤（如 grep 检查文档间关键字一致性）
- 4 份文档中 task 的定义措辞是否完全一致
- SCOE/指令接入作为新增 feature 在 4 份文档中的描述是否同步
