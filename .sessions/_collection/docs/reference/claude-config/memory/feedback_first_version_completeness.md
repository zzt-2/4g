重写不是从零开始，第一版应包含大部分功能，不采用多轮渐进式交付。

**Why:** 这是重写已有系统（不是新项目），需求已经明确，旧行为都是验证过的。用户明确表示"不打算太磨蹭"。

**How to apply:** 设计和实施时不要刻意砍功能分期交付。如果某功能有明确旧行为支撑且技术方案成熟，直接做到位。只有真正有技术不确定性的才考虑分期。
