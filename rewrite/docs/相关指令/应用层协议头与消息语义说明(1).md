# 应用层协议头与消息语义说明

## 文档目的

本文档用于给出当前 `master` 现役控制与状态协议的应用层头字段和消息语义。

本说明以当前 RTL 为准，主要依据：

- `import/rtl_source/ctrl_status_block/ctrl_status_block_pkg.sv`
- `import/rtl_source/ctrl_status_block/frame_processing/ctrl_status_cmd_param_splitter.sv`
- `import/rtl_source/ctrl_status_block/frame_processing/ctrl_status_tm_frame_builder.sv`

## 当前应用层头字

当前应用层头固定为 1 个 `32-bit word`：

```text
[31:28] version
[27:24] msg_type
[23:20] module_id
[19:12] group_id
[11:4]  param_count
[3:0]   reserve
```

对应共享定义：

- `HOST_CTRL_VERSION_W = 4`
- `HOST_CTRL_MSG_TYPE_W = 4`
- `HOST_CTRL_MODULE_ID_W = 4`
- `HOST_CTRL_GROUP_ID_W = 8`
- `HOST_CTRL_PARAM_COUNT_W = 8`
- `HOST_CTRL_RESERVE_W = 4`

## 当前消息类型

当前现役 `msg_type` 只保留两类：

| 数值 | 常量名 | 含义 |
| --- | --- | --- |
| `4'h1` | `HOST_MSG_REMOTE_CMD_C` | 主机下发遥控命令 |
| `4'h2` | `HOST_MSG_TELEMETRY_C` | 模块上报遥测帧 |

## 当前命令语义

当前 `master` 不再沿用旧版外部头字段模型或旧版消息分类作为现役公共头字段语义。

当前现役口径是：

- 公共头只负责说明“这是哪个模块、哪个组、后面有多少个 `32-bit` 参数字”
- 模块内部再通过本地 `group table / param table / action table` 把参数流落到真实配置项、枚举映射、脉冲请求或流注入
- 遥测方向同样只使用 `module_id + group_id + param_count`，不使用旧版外部字段模型

## 遥控 payload 模型

遥控接收路径为：

1. `ctrl_status_rs422_link_adapter` 输出 `host_cmd_axi4`
2. `ctrl_status_host_ctrl_core` 按 `module_id` 路由到目标一级模块
3. 目标模块内部用 `ctrl_status_cmd_param_splitter` 把整组命令拆成参数拍流
4. 再由 `ctrl_status_cmd_action_mapper` 转换为实现层动作
5. 最终由模块本地寄存器、脉冲或其他落地逻辑消费

因此，当前公共协议的最小语义单位是：

- `module_id`
- `group_id`
- `param_count`
- 参数字流

而不是旧模型里的“公共层直接识别对象号”。

## 遥测 payload 模型

遥测发送路径为：

1. `top_ctrl_status_block` 通过 `tm_req_valid / tm_req_group_id` 请求某模块某一组遥测
2. 模块本地 `tm_agent` 捕获 `cfg/stat` 快照
3. `ctrl_status_tm_frame_builder` 先发 1 个应用层头字，再顺序发出遥测数据字
4. `ctrl_status_host_ctrl_core` 汇聚各模块 `stat_axi4`
5. `ctrl_status_rs422_link_adapter` 再把状态流打包回 `RS422` 固定帧

当前遥测 `group_id` 只有两类共享定义：

- `8'h80 = TM_GROUP_RUNTIME_C`
- `8'h81 = TM_GROUP_CFG_C`

遥测 header 中的 `param_count` 表示数据区 word 数，不包含 header word。RS422 固定帧层看到的 payload 长度为 `(1 + param_count) * 4` 字节。

## 与 rs422-stat 分支应用层文档的关系

`rs422-stat` 分支中的应用层文档仍有参考价值，但主要作为：

- 历史需求来源
- 旧对象模型对照材料
- 旧上位机发送/观测语义草稿

不能直接视为当前 `master` 的现役协议合同。

当前 `master` 的现役应用层合同以本文件和 [应用层模块ID与遥测组说明.md](应用层模块ID与遥测组说明.md) 为准。
