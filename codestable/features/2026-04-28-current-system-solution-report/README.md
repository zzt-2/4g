# 当前系统方案设计报告资料

## 1. 写作口径

本文档目录用于沉淀《当前系统方案设计报告》的分片内容。报告面向甲方阅读，正文只讲系统功能、业务流程、运行环境、对接范围和图件说明，不展开代码结构、代码质量、内部改造路线或实现方案。

报告应按“最终系统已经具备什么、在测试现场如何运行、与上级系统如何协同、结果和文件如何形成并回传”的顺序组织。内部资料中涉及的旧任务系统、历史记录、外部指令码、northbound 等内容，只按功能事实转写，不把内部术语直接暴露成甲方主线。

正式正文只写已经完成、已经能按方案交付的功能。当前还没完成的能力可以在本目录中作为写作候选保留，但生成正式报告时必须删掉或暂不展开，不能用“可支持”“可配置”等模糊说法把未完成功能写成既有能力。本目录不设置“待确认事项”章节。

## 2. 报告范围

### 2.1 当前系统自身功能

这部分是报告的主体，应尽量写厚，体现系统不是单纯接口壳，而是已有本地测试能力的软件系统。

| 功能域 | 报告中应写的内容 |
| --- | --- |
| 本地操作界面 | 系统提供本地桌面化操作入口，支持功能导航、状态查看、参数配置、测试操作和历史数据查看。 |
| 设备与链路管理 | 支持串口、网络等本地通信链路配置，支持连接、断开、通信目标选择和链路状态显示。 |
| 协议帧与测试帧管理 | 支持按协议帧组织数据收发规则，承接测试数据解析、发送组包和结果展示。 |
| 数据接收 | 支持从本地测试设备或被测设备接收原始数据，并进入解析、展示、统计和记录流程。 |
| 数据解析与显示 | 支持将接收数据转换为可读参数、状态值、统计值或表达式结果，供界面显示和结果整理使用。 |
| 数据发送 | 支持手动发送、按目标发送、按帧发送、组合发送等本地控制能力。 |
| 自动发送能力 | 支持定时发送、触发发送、顺序发送等本地自动化发送能力；报告中只作为本地自动发送功能描述，不等同于上级系统下发的测试任务。 |
| 测试执行辅助 | 支持围绕测试过程进行开始、停止、状态观察、执行记录等操作，形成可追溯的测试过程。 |
| 历史记录与文件导出 | 支持保存过程数据、历史记录、CSV 或相关文件；报告中可作为结果整理的数据来源，不直接称为甲方 TestReport。 |
| 状态展示 | 支持运行状态、连接状态、发送状态、接收状态、异常状态等本地可视化展示。 |
| 外部指令能力 | 如需保留该类能力，统一写作外部指令码接收与控制能力，只说明指令识别、控制反馈、状态记录等通用能力，不出现具体历史项目名称。 |

### 2.2 面向甲方的对接功能候选

这部分按“二级子系统接入上级集成测试系统”的口径保留写作候选，用来防止后续分片漏项。正式报告只保留届时已经完成的对接能力；未完成的能力不写入正文，也不写成规划项。

| 对接能力 | 报告中应写的内容 |
| --- | --- |
| 子系统接入 | 完成后可写为：当前系统作为二级子系统接入上级集成测试系统，提供任务执行、状态反馈、结果回传等能力。 |
| 任务接收 | 完成后可写为：系统接收上级系统下发的测试任务及其相关用例、参数、执行要求。 |
| 测试启动 | 完成后可写为：系统根据上级系统指令启动测试任务或测试用例列表。 |
| 测试停止 | 完成后可写为：系统响应上级系统停止控制，并将停止后的状态和结果按约定回传。 |
| 状态查询 | 完成后可写为：上级系统可查询子系统运行状态、任务状态、链路状态和执行状态。 |
| 心跳保活 | 完成后可写为：系统与上级系统保持链路心跳，用于在线状态和通信健康判断。 |
| 用例结果回传 | 完成后可写为：系统在测试过程中或用例完成后形成单用例结果，并按约定回传。 |
| 任务结果回传 | 完成后可写为：系统在任务结束后形成任务级汇总结果。 |
| 文件回传 | 完成后可写为：系统将测试过程文件、结果文件或报告文件上传至约定位置，并通知上级系统。 |
| 测试报告回传 | 完成后可写为：系统形成结构化测试报告，表达任务、用例、结果、异常和文件索引等信息。 |
| 告警与异常反馈 | 完成后可写为：系统将设备异常、链路异常、执行异常、文件异常等信息反馈给上级系统。 |

### 2.3 不写入正文的内容

- 不写重写、重构、优化路线。
- 不写代码质量、模块边界、工程治理。
- 不写数据库 schema 或接口字段逐项定义。
- 不把本地串口 / 网络目标直接写成甲方设备编号。
- 不把历史记录 / CSV 直接写成 TestReport。
- 不把旧本地发送任务直接写成上级系统测试任务。
- 不把外部指令码接收与控制能力写成甲方当前 northbound 主线。

## 5. 图件规划

图件只服务正式正文。正式报告生成时，以 `figures/README.md` 和 `figures` 目录内已生成图片为准，不再要求补齐所有原计划图。

当前保留图片包括：系统范围思维导图、系统功能模块图、系统工作原理流程图、测试任务主流程图、部署拓扑图、接口结构图、测试业务对象 ER 图、测试任务单实体图、通信模式图、测试任务执行流程图、数据发送控制时序图、结果生成与报告材料流程图、实施安排甘特图。

未生成的辅助图不再作为正式正文必需图件；相关内容由正文段落覆盖。

## 6. 后续分片原则

- 每个分片优先写成可直接放进正式文档的正文，不写内部分析语气。
- 正式正文只写已经完成的功能；未完成的功能只留在本 README 的候选清单里，不进入分片正文。
- 每个章节先写一到两段功能说明，再按需要补业务流程和图，不为单个小功能继续拆标题。
- 章节文字可以适当概括，不追求逐项引用内部资料。
- 当前系统已有功能和甲方对接功能要分清：已有本地功能写成“系统具备”，已完成对接功能写成“系统提供/支持/对接”。
- 凡涉及历史记录、CSV、发送任务、外部指令码控制的内容，均按功能事实写，不替换甲方接口概念。
