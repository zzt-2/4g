# 当前系统方案设计报告图片清单

本文档用于集中管理《当前系统方案设计报告》的图片生成说明。图片生成后建议放在本目录下，文件名按“图号 + 英文短名”的方式命名，便于后续合稿时统一插入。

## 0. 统一制图规范

建议画布：横版 16:9，白底，适合 Word / WPS 正文插图。优先使用清晰的流程图、模块图、拓扑图、数据流图、时序图和业务对象图，不使用照片、三维透视、科技感背景或复杂装饰。

视觉风格：主系统模块使用浅蓝或蓝灰色，上级系统使用浅紫或灰蓝色，现场设备使用浅绿色，文件/结果材料使用浅黄色，异常/告警可使用浅橙色。线条使用深灰色，箭头清晰，所有文字使用规范中文。

文字原则：图中只保留短标签，不写长句。每个矩形节点尽量控制在 4 到 10 个汉字；复杂说明放在正文，不塞进图里。

生成注意：如果使用 AI 图片工具，文字可能不稳定，建议先生成“布局清楚的图”，再在 PPT、draw.io、ProcessOn、Visio 或 WPS 中手工覆盖文字。不要生成 Mermaid、PlantUML 或代码形式的图。

统一禁用：不要出现具体历史项目名称；不要出现代码、数据库表、类名、函数名；不要把本地串口 / TCP / UDP 通信目标画成上级系统设备编号；不要把历史记录或 CSV 画成正式测试报告本体。

## 1. 系统设计相关图片

### 图 1-1：系统范围思维导图

建议文件名：`fig-1-1-system-scope-mindmap.png`

图片类型：思维导图。

放置位置：`1.1 系统概述`。

画面布局：中心放一个较大的圆角矩形，文字为“当前系统”。四周按上、左、右、下或四象限展开四组一级分支，分别是“本地测试能力”“上级系统协同”“现场运行环境”“外部指令控制能力”。

节点内容：
- 本地测试能力：桌面界面、设备链路、协议帧、数据接收解析、数据发送、历史记录、文件导出。
- 上级系统协同：任务、状态、结果、文件材料。
- 现场运行环境：测试工作站、串口设备、网络设备、本地文件存储。
- 外部指令控制能力：指令识别、控制反馈、状态记录。

连线要求：中心节点向四组一级分支发散；一级分支再连接各自子节点。外部指令控制能力只作为旁支能力绘制，不画成上级系统主线。

图面重点：让甲方一眼看到系统不是单纯接口，而是有本地测试、设备链路、数据收发、结果文件和协同能力的完整执行端。

### 图 1-2：系统功能模块图

建议文件名：`fig-1-2-system-function-modules.png`

图片类型：模块图。

放置位置：`1.2 总体架构`。

左栏节点：上级集成测试系统、本地桌面操作界面。

中栏节点：协同接入、测试任务与执行控制、数据发送、数据接收、协议解析、状态告警、历史记录、结果整理、文件与报告材料、外部指令码接收与控制能力。

右栏节点：串口链路、TCP/UDP 网络链路、本地测试设备或被测设备、本地文件存储。

连线要求：上级集成测试系统和本地桌面操作界面都指向“协同接入 / 测试任务与执行控制”；执行控制分别指向数据发送和数据接收；数据发送连接串口链路和 TCP/UDP 网络链路；数据接收连接协议解析；协议解析回流到状态告警、历史记录和结果整理；结果整理连接文件与报告材料、本地文件存储和上级系统协同。

图面重点：突出“入口、执行、收发、解析、记录、结果材料”的功能闭环。外部指令码接收与控制能力画在中栏边侧，表示扩展接入能力。

画面布局：左侧上级系统，中间当前系统，右侧现场链路和设备，下方文件传输。

6. 当前系统记录发送结果。
7. 历史记录与状态展示更新发送状态和错误提示。

补充节点：触发发送场景中，在第一步之前增加“接收解析结果满足触发条件”。
